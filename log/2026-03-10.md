## What was built / fixed

### Copy HTML button rendering bug
- **Problem:** `badge_html` Copy HTML button in the viewer page was rendering raw JavaScript as visible text instead of executing on click
- **Root cause:** `JSON.stringify(badge)` was injected directly into an HTML `onclick="..."` attribute — the double quotes in the JSON string broke out of the attribute
- **Fix:** Moved badge string into a `<script>` block as `var BADGE_HTML`, button calls `copyBadge(this)` — no HTML escaping conflicts possible
- **File:** `src/routes/viewer.js`

### Member attribution bug
- **Problem:** Labels registered via Claude Chat (PL-000005, PL-000006) were landing as orphaned free-tier records — not attributed to Shelton's member account despite a valid API key
- **Root cause:** Claude Chat sends `Authorization: Bearer plg_live_...` — `authMember()` only checked `x-plgen-key` header, so the key was invisible to the member lookup
- **Fix:** Updated `src/auth.js` to also extract key from `Authorization: Bearer` header
- **File:** `src/auth.js`

### Backfill PL-000005 and PL-000006
- Updated `member_id` and `tier` directly in SQLite for the two orphaned labels
- All 6 labels now show correctly in the dashboard
